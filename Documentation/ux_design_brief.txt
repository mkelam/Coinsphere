## CRITICAL CONTEXT (Read First)

### Product Differentiation

This is **NOT** another portfolio tracker. Our core value proposition:

1. **AI Predictions** - Bull/bear forecasts with confidence scores (competitors don't have this)
2. **Risk Scoring** - Real-time "Degen Score" 0-100 for every asset
3. **99% Sync Accuracy** - Cross-validated pricing (vs competitors' 70-80%)
4. **Transparency** - Show users WHY predictions are made (not black-box)

**Design Implication:** Predictions and risk scores must be **primary UI elements**, not buried features.

### Target Users

- **Primary:** Active crypto traders (25-45, trades 10+ times/month, operates at night)
- **Device:** iPhone 14 Pro is primary device (design mobile-first)
- **Behavior:** Checks app 5-10x per day for quick portfolio health checks
- **Pain Point:** Can't predict when to enter/exit positions without our AI

## DESIGN SPECIFICATIONS

### Style Direction

**Glassmorphism Philosophy:**
- Glass effects **enhance data readability** (layer separation), not decoration
- Cards float on dark gradient background (depth hierarchy)
- Blur effects should **not reduce text contrast** (WCAG AA minimum)
- Use glass for: cards, modals, navigation bars
- **Don't use glass for:** buttons, alerts, critical information

**Minimalist Principles:**
- Information density > white space (financial app, not lifestyle brand)
- Remove element unless it serves data comprehension
- Every pixel earns its place through utility

**Dark Mode Only:**
- Crypto traders operate at night
- Reduces eye strain during long sessions
- Makes charts/numbers pop with higher contrast

## DESIGN CONSTRAINTS (Non-Negotiables)

### Information Hierarchy

1. **Portfolio value** → Primary (largest, highest contrast)
2. **AI predictions** → Secondary (cards, prominent placement)
3. **Risk scores** → Tertiary (visual indicators, color-coded)
4. **Holdings/transactions** → Supporting (tables, data-dense)

**Rule:** Every screen must answer "Am I making/losing money?" within 5 seconds

### Interaction Principles

- **2-click rule:** Any feature accessible in ≤2 taps
- **Real-time updates:** Prices refresh every 60s without full reload (WebSocket)
- **Pull-to-refresh:** Mobile gesture to force price sync
- **Skeleton loaders:** Show content structure while loading (never blank/spinner)

### Accessibility Requirements

- **Contrast:** WCAG AA minimum (4.5:1 for text, 3:1 for UI elements)
- **Keyboard nav:** All interactive elements focusable (tab order logical)
- **Screen readers:** Proper labels for charts, buttons, icons
- **Touch targets:** Minimum 44x44px (iOS/Android standard)

### Performance Constraints

- **Mobile-first:** Design for 375px width (iPhone SE) → scale up
- **Load time:** < 3s on 3G (use skeleton loaders, lazy load images)
- **Animation:** 60fps animations (use CSS transforms, not position/width)
- **Bundle size:** < 500KB initial page load

## ANIMATION & MICRO-INTERACTIONS

### Principles

- **Purposeful, not decorative** - animations clarify state changes
- **Fast (< 200ms)** - crypto traders want speed
- **Subtle** - don't distract from data

### Key Animations

**1. Portfolio Value Update**

```
Old value fades out → New value fades in (300ms ease)
Change % pulses green/red briefly (200ms)
```

**2. Card Hover**

```
Scale: 1 → 1.02 (150ms ease-out)
Shadow: light → medium (150ms)
Border: 0.1 → 0.15 opacity (150ms)
```

**3. Pull-to-Refresh**

```
User pulls down → spinner appears
Data loads → success checkmark (500ms)
Checkmark fades → new data animates in
```

**4. Skeleton Loading**

```
Show gray placeholders (shimmer effect)
Fade to real content when loaded (200ms)
```

**5. Risk Score Change**

```
Score increases: pulse green (300ms)
Score decreases: pulse red (300ms)
Gauge animates to new value (500ms ease)
```

**Portfolio loading:**


## RESPONSIVE BREAKPOINTS

### Mobile (375px - 767px)

- Single column layout
- Bottom navigation (Dashboard, Predictions, Risk, Settings)
- Collapsible sections (tap to expand)
- Cards full-width
- Tables convert to cards

### Tablet (768px - 1023px)

- Two-column layout where appropriate
- Side navigation (drawer on mobile)
- Cards 48% width (2 per row)
- Tables remain tables

### Desktop (1024px+)

- Three-column layout possible
- Persistent side navigation
- Cards 32% width (3 per row)
- Larger charts, more data visible

**Priority:** Design mobile first, scale up

## REFERENCE EXAMPLES

### Nail This Vibe ✅

- **Stripe Dashboard** - Data clarity, subtle depth, purposeful glass
- **Linear** - Minimalist, fast, keyboard-first, no fluff
- **Phantom Wallet** - Crypto-native, glassmorphic without excess
- **Revolut** - Financial data, scannable tables, clear hierarchy

### Avoid These ❌

- **Coinbase** - Rainbow gradients, overly colorful, distracting
- **Binance** - Information overload, cluttered, too much at once
- **Robinhood** - Gamification, confetti, not serious analytics
- **CoinMarketCap** - Outdated design, poor hierarchy, ad-heavy

## SUCCESS CRITERIA

### Design is successful if:

✅ User can assess portfolio health in **< 5 seconds** (timed test)  
✅ Predictions feel **trustworthy** (confidence scores visible, accuracy history shown)  
✅ Risk scores are **immediately actionable** (color-coded, clear danger signals)  
✅ Glass effects **enhance readability** (tested contrast ratios, not decoration)  
✅ Works **beautifully on iPhone 14 Pro** (primary user device)  
✅ Developers can implement **without asking questions** (components annotated)

### Design fails if:

❌ User must scroll to see portfolio value  
❌ Predictions look like "magic" (no transparency = no trust)  
❌ Glass effects reduce text contrast (fails WCAG AA)  
❌ Designers prioritized aesthetics over data density  
❌ Missing states (empty, loading, error not designed)

## FINAL NOTES

### What This Design Is

- **Functional** - Every pixel serves data comprehension
- **Trustworthy** - Transparency builds confidence in AI
- **Fast** - Optimized for quick portfolio checks
- **Professional** - Serious analytics, not a game

### What This Design Is NOT

- **Decorative** - No art project, serve the data
- **Overwhelming** - Complexity hidden behind progressive disclosure
- **Gimmicky** - No confetti, no gamification, no tricks

### Design Philosophy

> "Design is not just what it looks like and feels like. Design is how it works."  
> — Steve Jobs

For Coinsphere, **"how it works"** means: clarity, transparency, speed.
