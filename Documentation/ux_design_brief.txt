## CRITICAL CONTEXT (Read First)

### Product Differentiation

This is **NOT** another portfolio tracker. Our core value proposition:

1. **AI Predictions** - Bull/bear forecasts with confidence scores (competitors don't have this)
2. **Risk Scoring** - Real-time "Degen Score" 0-100 for every asset
3. **99% Sync Accuracy** - Cross-validated pricing (vs competitors' 70-80%)
4. **Transparency** - Show users WHY predictions are made (not black-box)

**Design Implication:** Predictions and risk scores must be **primary UI elements**, not buried features.

### Target Users

- **Primary:** Active crypto traders (25-45, trades 10+ times/month, operates at night)
- **Device:** iPhone 14 Pro is primary device (design mobile-first)
- **Behavior:** Checks app 5-10x per day for quick portfolio health checks
- **Pain Point:** Can't predict when to enter/exit positions without our AI

## DESIGN SPECIFICATIONS

### Style Direction

**Glassmorphism Philosophy:**
- Glass effects **enhance data readability** (layer separation), not decoration
- Cards float on dark gradient background (depth hierarchy)
- Blur effects should **not reduce text contrast** (WCAG AA minimum)
- Use glass for: cards, modals, navigation bars
- **Don't use glass for:** buttons, alerts, critical information

**Minimalist Principles:**
- Information density > white space (financial app, not lifestyle brand)
- Remove element unless it serves data comprehension
- Every pixel earns its place through utility

**Dark Mode Only:**
- Crypto traders operate at night
- Reduces eye strain during long sessions
- Makes charts/numbers pop with higher contrast

## DESIGN CONSTRAINTS (Non-Negotiables)

### Information Hierarchy

1. **Portfolio value** â†’ Primary (largest, highest contrast)
2. **AI predictions** â†’ Secondary (cards, prominent placement)
3. **Risk scores** â†’ Tertiary (visual indicators, color-coded)
4. **Holdings/transactions** â†’ Supporting (tables, data-dense)

**Rule:** Every screen must answer "Am I making/losing money?" within 5 seconds

### Interaction Principles

- **2-click rule:** Any feature accessible in â‰¤2 taps
- **Real-time updates:** Prices refresh every 60s without full reload (WebSocket)
- **Pull-to-refresh:** Mobile gesture to force price sync
- **Skeleton loaders:** Show content structure while loading (never blank/spinner)

### Accessibility Requirements

- **Contrast:** WCAG AA minimum (4.5:1 for text, 3:1 for UI elements)
- **Keyboard nav:** All interactive elements focusable (tab order logical)
- **Screen readers:** Proper labels for charts, buttons, icons
- **Touch targets:** Minimum 44x44px (iOS/Android standard)

### Performance Constraints

- **Mobile-first:** Design for 375px width (iPhone SE) â†’ scale up
- **Load time:** < 3s on 3G (use skeleton loaders, lazy load images)
- **Animation:** 60fps animations (use CSS transforms, not position/width)
- **Bundle size:** < 500KB initial page load

## RESPONSIVE BREAKPOINTS

### Mobile (375px - 767px)

- Single column layout
- Bottom navigation (Dashboard, Predictions, Risk, Settings)
- Collapsible sections (tap to expand)
- Cards full-width
- Tables convert to cards

### Tablet (768px - 1023px)

- Two-column layout where appropriate
- Side navigation (drawer on mobile)
- Cards 48% width (2 per row)
- Tables remain tables

### Desktop (1024px+)

- Three-column layout possible
- Persistent side navigation
- Cards 32% width (3 per row)
- Larger charts, more data visible

**Priority:** Design mobile first, scale up
